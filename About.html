DOCUMENTATION

1.1. Purpose

The purpose of the Pet Smart Manager project is to make pets owners an easier life managing all the details about them. You can save and edit memories of your pet, the medical history and more information about his health, like his meal plan for the entire week, and how to give him first aid in case of emergency. Also you can see other pets and details about them and also you can add them on your friend list to keep them close.


2.2.Product Functions

First of all, the users need to create an account and login in order to acces the application. Once you are logged in the app, you can create and edit your profile of your pet. Besides the details like age,gender, breed, name and others, you can use the other 3 categories: "Meal plan", "Multimedia" and "Medical history". Another page you can use within the application is dedicated for interaction and is named "Find friends".

2.3.User Classes and Characteristics

The main class that we anticipate to use this product is the owners of any kind of pet, the ones that love their animal and dedicate a lot of time for their health. Another class can be the veterinary doctors who will use this application to keep all the animals and their medical history under their atention.  

2.4. Operating Environment

A person who wants to use this application needs to have any device that has internet connection and can use a web browser.

2.5.Design and Implementation Constraints

One of the limitations of the design of the applications is the fact that the app is english-only and can be hard to use if you don't know the language too well. Other than that, another thing is that you can't chat with other pets and also you can't see the multimedia of other pets.

2.6.User Documentation

The application is pretty suggestive and simple and we have the instruction on every button that you can press to change the action of every page you want to proceed.Also the users who have any questions can contact us by email.

3.1. User Interfaces

After you log in, the navigation bar is shown on the left of the screen on every page helping users to acces all the features the application has. The nav-bar is responsive and if the users acces the app from a phone or tablet, there will be an icon on the top-right of the screen to open nav-bar. 

The navigation bar contains:

  1. About
  2. Find Friends
  3. Profile: Meal Plan, Multimedia, Medical History
  4. Sign out

3.2. Hardware Interfaces

The application is based on HTML, CSS and vanilla JAVASCRIPT and can be used on any browser

3.3 Software Interfaces

We developed this app on windows using visual studio code or any text editor and wrote HTML, CSS, JS code. We store the data using XAMPP and MYSQL.

3.4 Communications Interfaces

The communications functions you need for this product is e-mail, network connection, a device that can use a web browser, an web browser. The communication standard that is used is HTTP.


4.1. "Login" feature

This is a high priority feature as you need to login using your email and password to acces the application.

4.2. "Sign-up" feature

The users who don't have an account can create one using this feature. "Sign-up" is a high priority feature. 

4.3. "About" feature

On the about feature you can see the documentation of the application and also details about the project developers like name and e-mail. This one is low priority.

4.4. Profile features

Multimedia, Meal Plan and Medical History are medium features and are not used every time and they are not necessary.

5.3. Security Requirements

The users create an account using his email, name and password. Other users can only see the details that are public. The multimedia, meal plan and medical history is private for any user.
